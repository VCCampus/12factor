# 文档目录

此目录包含项目的所有文档、规划和研究资料。

## 目录结构

### 📋 `/plans/` - 项目规划文档
项目实施计划、技术方案和配置文件
- `web3scv8_v4.toml` - 主配置文件（当前版本4.0.0）
- 各阶段实施计划和技术路线图
- TOML配置格式规范和迁移方案

### 📊 `/reports/` - 实施报告
项目各阶段的开发报告和成果总结
- 按日期命名格式：`yymmdd_{任务描述}.md`
- 包含技术实施细节、问题解决方案和优化建议
- 最新报告：`250905_nav_simplification.md` - 导航栏简化实施报告

### 🔍 `/research/` - 研究资料
外部参考资料、用户调研和技术调研
- CSS数字创业核心知识体系文档
- 面试题库和问卷调研
- 设计参考和技术调研资料

## 文档编写规范

### 命名约定
- 计划文档：`{功能名}_plan.md` 或 `{日期}_{功能描述}.md`
- 报告文档：`yymmdd_{任务总结}.md`
- 研究文档：按主题分类，使用描述性名称

### 内容结构
每个文档应包含：
1. 目标和背景
2. 技术方案或实施步骤
3. 关键决策和理由
4. 实施结果和验证
5. 后续优化建议

## 使用指南

### 查找文档
- 最新项目状态：查看 `/reports/` 中的最新报告
- 技术配置：查看 `/plans/web3scv8_v4.toml`
- 历史决策：查看对应阶段的规划和报告文档

### 添加新文档
1. 确定文档类型（规划/报告/研究）
2. 选择合适的子目录
3. 遵循命名约定
4. 包含完整的上下文和决策依据
